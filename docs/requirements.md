# 요구사항 명세서

## 1. 서비스 목표

사용자가 AI와 짧은 대화를 나누는 것만으로 자신의 여행 취향을 파악하고, 이번 달 뜨는 관광지와 혼잡을 피하는 날짜·숙박을 조합한 맞춤 패키지를 즉시 받아볼 수 있게 한다.

---

## 2. 사용자 스토리

### 비회원
- 서비스 소개 페이지를 보고 AI 취향 탐색을 체험할 수 있다.
- 패키지 결과는 1건만 미리보기로 확인할 수 있다. (나머지는 로그인 유도)

### 회원
- AI와 대화해 취향 프로필을 생성·수정할 수 있다.
- 취향 기반 맞춤 패키지 목록(최대 5개)을 받아볼 수 있다.
- 패키지 상세(관광지·숙박·날짜·혼잡도)를 확인할 수 있다.
- 마음에 드는 패키지를 저장(북마크)할 수 있다.
- 저장한 패키지 목록을 마이페이지에서 관리할 수 있다.

## 3. 기능 요구사항

### 3.1 인증 (Auth)
| ID | 요구사항 |
|----|----------|
| A-01 | 이메일 + 비밀번호 회원가입 |
| A-02 | 소셜 로그인 (Google, Kakao) |
| A-03 | 로그인 상태 유지 (Supabase 세션) |
| A-04 | 비밀번호 재설정 (이메일 링크) |

### 3.2 AI 취향 탐색 대화 (Chat)
| ID | 요구사항 |
|----|----------|
| C-01 | 서비스가 먼저 질문을 시작한다 (사용자가 빈 입력창을 볼 일 없음) |
| C-02 | 한 세션에서 질문 5~8개로 취향 파악 완료 |
| C-03 | 수집 항목: 동행 유형, 선호 자연환경, 활동 강도, 혼잡 회피 여부, 반려동물 동반, 웰니스/힐링 선호, 캠핑·글램핑 관심도 |
| C-04 | Claude API 스트리밍으로 답변을 실시간 표시 |
| C-05 | 대화 종료 후 취향 프로필을 DB에 저장 |
| C-06 | 기존 취향 프로필이 있는 회원은 "이전 취향으로 추천" 또는 "다시 탐색" 선택 가능 |
| C-07 | 대화 도중 이탈 시 진행 상황 유지 (세션 내) |

### 3.3 관광 빅데이터 트렌드 감지
| ID | 요구사항 |
|----|----------|
| B-01 | 매일 1회 관광 빅데이터 API를 호출해 이번 달 급상승 관광지 목록을 갱신 |
| B-02 | 급상승 순위 상위 20개 지역을 내부 DB에 캐싱 |
| B-03 | 캐싱 데이터 갱신 실패 시 이전 데이터를 유지하고 알림 로그 기록 |

### 3.4 패키지 자동 생성
| ID | 요구사항 |
|----|----------|
| P-01 | 취향 프로필 + 트렌드 관광지를 조합해 후보 관광지 목록 생성 |
| P-02 | 관광지 집중률 예측 API로 향후 30일 중 혼잡도 하위 날짜 3개 선정 |
| P-03 | 선정 날짜 기준 인근 숙박(호텔·펜션·캠핑) 데이터 조회 |
| P-04 | 반려동물 동반 시 동반 가능 시설만 필터링 |
| P-05 | 웰니스 선호 시 웰니스관광정보 API 데이터 포함 |
| P-06 | 최종 패키지 최대 5개 생성, 각 패키지에 관광지·숙박·추천 날짜·혼잡도 정보 포함 |
| P-07 | 패키지 생성 소요 시간 목표: 10초 이내 |
| P-08 | 외부 API 일부 실패 시 가용 데이터로 부분 패키지 생성 (전체 실패 방지) |

### 3.5 패키지 조회·상세
| ID | 요구사항 |
|----|----------|
| V-01 | 패키지 목록: 썸네일·관광지명·추천 날짜·혼잡도 요약 카드 표시 |
| V-02 | 패키지 상세: 관광지 소개, 숙박 옵션, 혼잡도 달력, 지도(카카오맵 또는 네이버지도) |
| V-03 | 패키지 북마크 저장/해제 (회원 전용) |
| V-04 | 마이페이지에서 저장 패키지 목록 조회·삭제 |

### 3.6 비기능 요구사항
| ID | 요구사항 |
|----|----------|
| N-01 | 모바일 우선 반응형 UI (360px ~ 1280px) |
| N-02 | 핵심 페이지 LCP 3초 이내 (Vercel Edge 활용) |
| N-03 | 외부 API 응답 캐싱으로 중복 호출 최소화 |
| N-04 | Supabase RLS로 사용자 데이터 격리 |
| N-05 | 환경 변수로 API 키 관리, 코드 내 하드코딩 금지 |

## 4. 제외 범위 (Out of Scope)

- 실시간 예약·결제 기능 (외부 여행사 링크 연결로 대체)
- 리뷰·평점 시스템
- 소셜 공유 기능
- 관리자 대시보드
- 다국어 지원 (한국어 전용)
